		\subsection{Streamkonzept}
			\begin{compactitem}
				\item Ein \lc{Stream} repräsentiert einen sequentiellen Datenstrom.
				\item Die Operatoren auf dem \lc{Stream} sind \lc{<}\lc{<} und \lc{>}\lc{>}.\newline
				Für vordefinierte Datentypen sind diese Operatoren schon definiert, für eigene selbstdefinierte Klassen können diese Operatoren überladen werden).
				\item C++ stellt 4 Standardstreams zur Verfügung:
					\begin{compactitem}
						\item \lc{cin}: Standard-Eingabesteam, normalerweise die Tastatur
						\item \lc{cout}: Standard-Ausgabestream, normalerweise der Bildschirm
						\item \lc{cerr}: Standard-Fehlerausgabestream, normalerweise der Bildschirm
						\item \lc{clog}: mit \lc{cerr} gekoppelt
					\end{compactitem}
				\item Alle diese Streams können auch mit einer Datei verbunden werden.	
			\end{compactitem} 
	\end{minipage}

	\subsection{Streams und Dateien \verweiscpp{A.5}}
		Streams sind Abstraktionen für die zeichenweise Ein-/Ausgabe. Dabei ist es unerheblich, ob die Ausgabe auf den Bildschirm erfolgt oder auf eine Datei. Dateien können also über die normalen Möglichkeiten der Ein-/Ausgabe beschrieben werden.
		\begin{compactitem}
			\item Dateien müssen geöffnet werden. Das Öffnen einer Datei assoziiert die physikalische Datei mit dem enstprechenden Stream.
			\item Dateien müssen nach ihrer Verwendung geschlossen werden. Jede Datei wird automatisch geschlossen, wenn der assoziierte Stream geschlossen wird.
			\item Die Verarbeitung von Dateien kann im Textmodus oder im Binärmodus erfolgen.
			\begin{compactitem}
				\item Binärmodus: Streams werden Zeichen für Zeichen eingelesen/geschrieben ohne jegliche Transformation.
				\item Textmodus: Im Textmodus werden einzelne Zeichen umgewandelt. So wird zum Beispiel $endl$ in die plattformübliche Zeilenende-Sequent ($0x0A$ $0x0D$)
			\end{compactitem}
		\end{compactitem}

		\subsubsection{Lesen und Setzen von Positionen \verweiscpp{A.5.2}}
			Die aktuelle Position innerhalb einer Datei kann beliebig verändert werden. Dazu stehen folgende Typen und Methoden zur Verfügung:
			\begin{compactitem}
				\item $streampos$ ist der Typ einer Dateiposition.
				\item $seekg(offset,direction)$ zum Beispiel setzt die aktuelle Dateiposition einer Datei. $offset$ gibt die Position vom Dateianfang bzw. -ende aus an, $direction$ legt fest, von wo aus die Position bestimmt wird.
				\begin{compactitem}
					\item $ios::beg$: Dateianfang
					\item $ios::cur$: Aktuelle Position
					\item $ios::end$: Dateiende
				\end{compactitem}
				\item $tellg$ liefert die aktuelle Position in der Datei.
				\item $seekp$ und $tellp$ sind die entsprechenden Versionen für die Put-Varianten.
			\end{compactitem}
